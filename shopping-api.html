1 파워링크 API와의 차이점

💡 인증 방법, Base URL, 헤더 구조는 파워링크 API와 완전히 동일합니다. 캠페인 세팅 단계부터 쇼핑검색광고 고유의 설정이 필요합니다.
항목파워링크쇼핑검색광고
Base URLhttps://api.searchad.naver.com동일
인증 방식HMAC-SHA256동일
campaignTpWEB_SITESHOPPING
Ad typeTEXT_45SHOPPING_PRODUCT_AD
키워드 등록직접 등록 (POST /ncc/keywords)시스템 자동매칭 — 별도 등록 없음
소재 기반제목+설명 텍스트 직접 입력CPS 입점을 통해 연동된 EP 상품 활용
💡 EP란? 네이버 쇼핑 CPS 입점 시 연동되는 상품 피드(상품명·가격·카테고리·속성·이미지 등)입니다. 쇼핑검색광고는 이미 입점 과정에서 연동된 EP 상품 데이터를 광고 소재로 활용합니다. EP를 별도로 새로 연동할 필요는 없습니다.
⚠️ 쇼핑검색광고 전용 EP를 별도로 운영하고 싶다면?
가격비교와 독립적으로 쇼핑검색광고를 운영하고 싶은 경우, 쇼핑검색광고 전용 EP를 별도로 생성하여 활용하는 방식을 선택할 수 있습니다.

2 사전 준비 — Business Channel & Product Group 조회

캠페인 세팅 전, 광고 계정에 연결된 쇼핑 채널과 상품 그룹을 API로 먼저 조회해야 합니다.

2-1. Business Channel 조회 — GET /ncc/channels

광고 계정에 연결된 쇼핑 채널(CPS 입점 완료 상태)을 조회합니다.

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/channels" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

응답 예시:

json 
[
  {
    "nccBusinessChannelId": "ch-001",
    "channelTp": "SITE",
    "name": "내 쇼핑몰",
    "pcInspectStatus": "APPROVED"
  }
]
⚠️ pcInspectStatus가 APPROVED인 채널만 광고에 사용할 수 있습니다.
응답의 nccBusinessChannelId를 메모해 두세요 — 광고그룹 생성 시 pcChannelId / mobileChannelId 값으로 사용됩니다.

2-2. Product Group 조회 — GET /ncc/product-groups

CPS 입점 시 연동된 EP 상품들은 상품 그룹(Product Group) 단위로 관리됩니다. 광고그룹 생성 전, 계정에 등록된 상품 그룹 목록을 조회하여 nccProductGroupId를 확인합니다.

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/product-groups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

응답 예시:

json 
[
  {
    "nccProductGroupId": "pg-001",
    "nccBusinessChannelId": "ch-001",
    "mallId": "my-mall-001",
    "mallName": "내 쇼핑몰",
    "name": "전체 상품 그룹",
    "brandName": "My Brand",
    "registeredProductType": "GENERAL",
    "registrationMethod": "WHOLE",
    "numberOfAdgroups": 0
  }
]
필드설명
nccProductGroupId상품 그룹 고유 ID — 광고그룹 생성 시 사용
nccBusinessChannelId연결된 Business Channel ID
mallId / mallName쇼핑몰 식별 정보
name상품 그룹 이름
registeredProductTypeGENERAL (일반 상품) / CATALOG (카탈로그) / BLOG
registrationMethodWHOLE (전체 등록) / EACH (선택 등록)
numberOfAdgroups이 상품 그룹을 사용 중인 광고그룹 수
💡 nccProductGroupId는 광고그룹 생성 시 연결할 상품 그룹을 지정하는 데 사용됩니다.

3 핵심 API 워크플로우 — 단계별 세팅

쇼핑검색광고 API 세팅 권장 순서:

사전 조회 Channel ID + Product Group ID 확인
→
STEP 1 캠페인 생성
→
STEP 2 광고그룹 생성
→
STEP 3 소재(상품) 등록
💡 키워드 등록 단계 없음 — EP 기반 시스템 자동매칭

STEP 1. 캠페인 생성 — POST /ncc/campaigns

파워링크와 동일한 엔드포인트입니다. campaignTp 값만 SHOPPING으로 다릅니다.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "campaignTp": "SHOPPING",
    "name": "쇼핑검색광고_캠페인",
    "dailyBudget": 100000,
    "useDailyBudget": true
  }'
파라미터설명쇼핑 설정값
campaignTp캠페인 유형"SHOPPING" (파워링크는 "WEB_SITE")
name캠페인 이름자유 입력
dailyBudget하루 예산 (원 단위)예: 100000
useDailyBudget하루 예산 사용 여부true

응답에서 확인할 것:

json 
{
  "nccCampaignId": "cmp-shop-001"
}

STEP 2. 광고그룹 생성 — POST /ncc/adgroups

SKU 규모에 따라 그룹핑 전략이 달라집니다.

Case 1 — 카테고리별 그룹핑 (SKU 많은 경우)

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-shop-001",
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}",
    "nccProductGroupId": "pg-001",
    "name": "카테고리1_그룹",
    "bidAmt": 200
  }'

Case 2 — 핵심 키워드별 그룹핑 (SKU 적은 경우)

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-shop-001",
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}",
    "nccProductGroupId": "pg-001",
    "name": "핵심키워드1_그룹",
    "bidAmt": 200
  }'
파라미터설명비고
nccCampaignIdSTEP 1에서 받은 캠페인 IDRequired
pcChannelIdPC 비즈니스 채널 IDRequired — CHAPTER 2-1 응답의 nccBusinessChannelId
mobileChannelId모바일 비즈니스 채널 IDRequired — 동일 값 사용 가능
nccProductGroupId상품 그룹 IDCHAPTER 2-2 응답의 nccProductGroupId
name광고그룹 이름그룹핑 전략에 맞게 명명
bidAmt기본 입찰가 (원 단위, 70~100,000)소재별 개별 설정도 가능

응답에서 확인할 것:

json 
{
  "nccAdgroupId": "grp-shop-001"
}

STEP 3. 소재(상품) 등록 — POST /ncc/ads

쇼핑검색광고의 소재는 CPS 입점 시 연동된 EP 상품을 기반으로 생성됩니다. type을 SHOPPING_PRODUCT_AD로 지정하여 등록합니다. 소재별 입찰가는 adAttr의 bidAmt로 설정하며, 그룹 기본 입찰가를 따르려면 useGroupBidAmt: true로 설정합니다.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/ads" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccAdgroupId": "grp-shop-001",
    "type": "SHOPPING_PRODUCT_AD",
    "adAttr": {
      "bidAmt": 200,
      "useGroupBidAmt": false
    }
  }'
파라미터설명비고
nccAdgroupId광고그룹 IDSTEP 2에서 획득
type소재 유형"SHOPPING_PRODUCT_AD" 고정 (GitBook 확인, GET /ncc/product-groups)
adAttr.bidAmt소재별 개별 입찰가개별 입찰가 설정 시 사용
adAttr.useGroupBidAmt그룹 기본 입찰가 사용 여부true이면 그룹 입찰가를 따름 (GitBook 확인)
💡 키워드는 별도로 등록하지 않습니다. EP 상품의 소재명·속성·카테고리 정보를 기반으로 시스템이 자동으로 관련 키워드에 매칭합니다.
⚠️ 추가 홍보문구 등 소재 상세 커스텀 필드는 GitBook Reference에 SHOPPING_PRODUCT_AD 타입의 ad 오브젝트 스키마가 별도 문서화되어 있지 않습니다. 상세 구현 시 네이버 검색광고 API 공식 가이드를 확인하세요.

4 소재 조회 및 수정

소재 전체 조회 — GET /ncc/ads

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/ads?nccAdgroupId=grp-shop-001" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

소재 입찰가 수정 — PUT /ncc/ads/{nccAdId}

bash / curl 
curl -X PUT "https://api.searchad.naver.com/ncc/ads/{nccAdId}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "adAttr": {
      "bidAmt": 300,
      "useGroupBidAmt": false
    }
  }'

5 운영 API — 조회·수정·통계

캠페인/그룹 상태 관리

작업Method엔드포인트
캠페인 전체 조회GET/ncc/campaigns
캠페인 일시정지PUT/ncc/campaigns/{id}
광고그룹 조회GET/ncc/adgroups

캠페인 일시정지 예시:

bash / curl

  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{"userLock": true}'

광고그룹 입찰가 수정 예시:

bash / curl

  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \

통계 조회 — GET /stats

파워링크와 동일한 엔드포인트를 사용합니다.

bash / curl 
curl -X GET "https://api.searchad.naver.com/stats?id={캠페인ID 또는 그룹ID 또는 소재ID}&fields=[\"clkCnt\",\"impCnt\",\"salesAmt\",\"convCnt\"]&timeRange={\"since\":\"2026-04-01\",\"until\":\"2026-04-10\"}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

쇼핑검색광고에서 주요하게 볼 지표:

지표 필드명의미쇼핑에서 중요한 이유
impCnt노출수자동매칭 범위 확인
clkCnt클릭수소재 CTR 판단
salesAmt광고비예산 소진 확인
ctr클릭률소재 매력도 판단
convCnt전환수실구매 연결 여부
convAmt전환 매출액ROAS 계산 기반

대용량 통계 — /stat-reports

조회 기간이 길거나 데이터 양이 많을 때 비동기 리포트 방식을 사용합니다:

흐름 
Step 1. POST /stat-reports  → 리포트 생성 요청 → reportId 획득
Step 2. GET  /stat-reports/{reportId}  → 상태 확인 (WAITING → RUNNING → DONE)
Step 3. DONE 상태 → 리포트 다운로드 URL에서 CSV 파일 다운로드

6 API 운영 팁 — 쇼핑검색광고 특화

소재 품질이 곧 노출 범위

쇼핑검색광고는 키워드를 직접 등록하지 않으므로, 상품 정보의 품질이 노출 범위를 결정합니다:

  • 상품명에 핵심 키워드를 풍부하게 포함 → 더 많은 키워드에 자동 매칭
  • 카테고리를 정확하게 설정 → 관련 검색어 매칭 정확도 향상
  • 추가 홍보문구는 항상 입력 → CTR 향상의 가장 빠른 방법

그룹별 입찰가 분리 관리

SKU가 많은 경우, 그룹별로 입찰가를 다르게 설정하여 예산을 효율적으로 운영합니다:

bash / curl 
# 고마진 카테고리 그룹 → 높은 입찰가
curl -X PUT "https://api.searchad.naver.com/ncc/adgroups/{고마진그룹ID}" \
  -d '{"bidAmt": 500}'

# 일반 카테고리 그룹 → 낮은 입찰가
curl -X PUT "https://api.searchad.naver.com/ncc/adgroups/{일반그룹ID}" \
  -d '{"bidAmt": 150}'
