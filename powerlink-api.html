1 사전 준비 — API 라이선스 발급

필수 조건 체크리스트

항목내용
네이버 광고 계정사이트검색광고(파워링크) 캠페인이 하나 이상 존재해야 함
API 라이선스광고 시스템 내에서 별도 신청 필요
발급 정보API KEY (X-API-KEY), SECRET KEY (서명 생성용), Customer ID (X-Customer)

API 라이선스 발급 단계

  1. 네이버 광고 시스템(searchad.naver.com) 로그인
  2. 우측 상단 [내 정보] → [API 관리] 클릭
  3. [API 라이선스 신청] 버튼 클릭
  4. 이용 약관 동의 후 신청 완료
  5. API KEY / SECRET KEY 화면에서 확인 및 저장
⚠️ SECRET KEY는 발급 시 한 번만 표시됩니다. 반드시 안전한 곳에 저장하세요.

2 API 기본 구조

항목값
Base URLhttps://api.searchad.naver.com
방식REST API
데이터 포맷JSON
인코딩UTF-8

전체 요청 구조

구조도 
https://api.searchad.naver.com/{리소스 경로}
  └── HTTP Method: GET / POST / PUT / DELETE
  └── Headers: 인증 정보 4개 (필수)
  └── Body: JSON (POST/PUT 요청 시)

3 인증 (Authentication)

필수 헤더 4가지

모든 API 요청에 아래 4개 헤더를 반드시 포함해야 합니다:

헤더명설명
X-Timestamp현재 시각 (Unix timestamp, 밀리초)
X-API-KEY발급받은 API KEY
X-Customer광고주 Customer ID
X-SignatureHMAC-SHA256 서명값

서명(Signature) 생성 공식

서명 공식 
서명 대상 문자열 = {timestamp} + "." + {HTTP_METHOD} + "." + {request_uri}

X-Signature = Base64( HMAC-SHA256( SECRET_KEY, 서명 대상 문자열 ) )

예시:

예시 
timestamp = 1712800000000
method    = GET
uri       = /ncc/campaigns

서명 대상 = "1712800000000.GET./ncc/campaigns"
X-Signature = Base64(HMAC-SHA256(SECRET_KEY, "1712800000000.GET./ncc/campaigns"))

curl 실제 요청 예시

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: 1712800000000" \
  -H "X-API-KEY: {발급받은 API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {생성한 서명값}" \
  -H "Content-Type: application/json"

응답 예시 (200 OK):

json 
[
  {
    "nccCampaignId": "cmp-001",
    "campaignTp": "WEB_SITE",
    "name": "브랜드 캠페인",
    "status": "ELIGIBLE",
    "dailyBudget": 100000
  }
]
💡 입문자 팁: curl은 별도 프로그램 설치 없이 Mac/Linux 터미널, Windows PowerShell에서 바로 사용 가능합니다.
Java / PHP / C# 샘플 코드
다른 언어 샘플은 GitBook 공식 문서(naver.github.io/searchad-apidoc)를 참고하세요.

사전 준비. Business Channel 조회 — GET /ncc/channels

광고그룹 생성 시 필수인 pcChannelId / mobileChannelId 값을 먼저 확인합니다.

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/channels" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"
💡 응답에서 channelTp가 SITE인 채널의 nccBusinessChannelId를 메모해 두세요. 이 값을 STEP 2의 pcChannelId / mobileChannelId에 사용합니다.

STEP 1. 캠페인 생성 — POST /ncc/campaigns

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "campaignTp": "WEB_SITE",
    "name": "내 브랜드 캠페인",
    "dailyBudget": 100000,
    "useDailyBudget": true
  }'
파라미터설명예시값
campaignTp캠페인 유형 (파워링크 = WEB_SITE 고정)"WEB_SITE"
name캠페인 이름"브랜드 캠페인"
dailyBudget하루 예산 (원 단위)100000
useDailyBudget하루 예산 사용 여부true

응답에서 확인할 것:

json 
{
  "nccCampaignId": "cmp-abc123"
}

STEP 2. 광고그룹 생성 — POST /ncc/adgroups

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-abc123",
    "name": "브랜드키워드_그룹",
    "bidAmt": 1000,
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}"
  }'
💡 pcChannelId와 mobileChannelId는 Business Channel API(GET /ncc/channels)에서 조회한 nccBusinessChannelId 값을 사용합니다. WEB_SITE 캠페인에서는 SITE 타입 채널 ID를 입력합니다.
파라미터설명비고
nccCampaignIdSTEP 1에서 받은 캠페인 IDRequired
name광고그룹 이름Required
pcChannelIdPC 비즈니스 채널 IDRequired — GET /ncc/channels 응답의 nccBusinessChannelId
mobileChannelId모바일 비즈니스 채널 IDRequired — 동일 값 사용 가능
bidAmt기본 입찰가 (원 단위, 70~100,000)미입력 시 기본값 70

STEP 3. 소재 등록 — POST /ncc/ads

소재 내용은 최상위가 아닌 "ad" 오브젝트 안에 중첩하여 전달합니다.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/ads" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccAdgroupId": "grp-xyz456",
    "type": "TEXT_45",
    "ad": {
      "headline": "공식 스토어 최대 30% 할인",
      "description": "지금 구매하면 무료배송. 오늘만 특가!",
      "pc": {
        "final": "https://www.brandstore.com/sale"
      },
      "mobile": {
        "final": "https://m.brandstore.com/sale"
      }
    }
  }'
파라미터설명제약
nccAdgroupId광고그룹 ID (Required)STEP 2에서 획득
type소재 유형 (Required)"TEXT_45" 고정
ad.headline광고 제목1~15자
ad.description광고 설명문20~45자
ad.pc.finalPC 연결 URLhttps:// 또는 http://
ad.mobile.final모바일 연결 URLhttps:// 또는 http://

STEP 4. 키워드 등록 — POST /ncc/keywords

nccAdgroupId는 request body가 아닌 URL query parameter로 전달합니다. Body는 키워드 객체의 배열입니다.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/keywords?nccAdgroupId=grp-xyz456" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "keyword": "나이키 운동화",
      "bidAmt": 500,
      "useGroupBidAmt": false
    },
    {
      "keyword": "나이키 에어포스 할인",
      "bidAmt": 300,
      "useGroupBidAmt": false
    }
  ]'
파라미터위치설명비고
nccAdgroupIdQuery param키워드를 등록할 광고그룹 IDSTEP 2에서 획득
keywordBody등록할 키워드Required, 공백 포함 가능
bidAmtBody키워드별 개별 입찰가 (70~100,000)useGroupBidAmt: false 시 적용
useGroupBidAmtBody그룹 기본 입찰가 사용 여부true면 그룹 입찰가 따름
linksBody키워드별 개별 랜딩 URLOptional

💡 키워드별 개별 랜딩 URL 설정

파워링크는 키워드마다 다른 랜딩 페이지를 지정할 수 있습니다. 키워드 등록 시 선택적(Optional) 필드인 links에 PC/모바일 URL을 각각 설정하면 되며, 설정하지 않을 경우 광고소재(Ad)에 설정된 URL이 기본으로 사용됩니다.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/keywords?nccAdgroupId=grp-xyz456" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "keyword": "나이키 운동화 세일",
      "bidAmt": 500,
      "useGroupBidAmt": false,
      "links": {
        "pc": { "final": "https://www.brandstore.com/shoes/sale" },
        "mobile": { "final": "https://m.brandstore.com/shoes/sale" }
      }
    }
  ]'
💡 links의 구조는 STEP 3 광고소재 등록의 ad.pc.final / ad.mobile.final과 동일한 패턴입니다.

통계 조회 — GET /stats

광고 성과 데이터를 API로 가져오는 핵심 기능입니다.

bash / curl 
curl -X GET "https://api.searchad.naver.com/stats?id={리소스ID}&fields=[\"clkCnt\",\"impCnt\",\"salesAmt\"]&timeRange={\"since\":\"2026-04-01\",\"until\":\"2026-04-10\"}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"
지표 필드명의미
impCnt노출수

  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

응답 데이터:

json 
{

필드의미

  -H "X-Signature: {signature}" \

schedule요일·시간대 노출 스케줄Required
⚠️ 확장소재 콘텐츠(서브링크 URL 등) 상세 포맷은 GitBook FAQ를 참조하세요: github.com/naver/searchad-apidoc/wiki/FAQ

7 API 운영 팁

공유예산 활용 — /ncc/shared-budgets

여러 캠페인의 예산을 하나의 공유 예산으로 통합 관리합니다:

구조도 
POST /ncc/shared-budgets  → 공유예산 생성
PUT  /ncc/campaigns/{id}  → sharedBudgetId 연결

여러 캠페인이 하나의 예산 풀을 공유 → 효율적인 예산 소진, 한 곳에서 관리

API 호출 시 주의사항

항목내용
Timestamp 유효 시간서버 시간 기준 ±300초(5분) 이내
요청 제한 (Rate Limit)초당 요청 수 제한 있음 — 과도한 호출 시 429 오류
배열 등록 한도키워드 등 배열형 POST 시 1회 최대 100개 권장
Customer ID헤더의 X-Customer는 숫자 형태의 광고 계정 ID

주요 에러 코드

HTTP 코드의미해결 방법
400잘못된 요청 (파라미터 오류)요청 Body/파라미터 확인
401인증 실패Signature 생성 로직 재확인
403권한 없음Customer ID 확인
404리소스 없음ID 값 확인
429호출 한도 초과요청 빈도 줄이기
500서버 오류잠시 후 재시도
Java / PHP / C# 샘플 코드
다른 언어 샘플은 GitBook 공식 문서(naver.github.io/searchad-apidoc)를 참고하세요.
