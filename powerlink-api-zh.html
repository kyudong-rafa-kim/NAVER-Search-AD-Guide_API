1 前期准备 — API授权申请

必要条件核查清单

项目内容
Naver广告账户需要至少存在一个网站搜索广告（Powerlink）广告系列
API授权需要在广告系统内单独申请
授权信息API KEY（X-API-KEY）、SECRET KEY（用于生成签名）、Customer ID（X-Customer）

API授权申请步骤

  1. 登录Naver广告系统（searchad.naver.com）
  2. 点击右上角 [我的信息] → [API管理]
  3. 点击 [申请API授权] 按钮
  4. 同意服务条款后完成申请
  5. 在页面上查看并保存API KEY / SECRET KEY
⚠️ SECRET KEY仅在授权时显示一次。请务必保存在安全的地方。

3 认证（Authentication）

4个必填请求头

所有API请求必须包含以下4个请求头：

请求头名称说明
X-Timestamp当前时间（Unix timestamp，毫秒）
X-API-KEY已申请的API KEY
X-Customer广告主Customer ID
X-SignatureHMAC-SHA256签名值

签名（Signature）生成公式

签名公式 
签名目标字符串 = {timestamp} + "." + {HTTP_METHOD} + "." + {request_uri}

X-Signature = Base64( HMAC-SHA256( SECRET_KEY, 签名目标字符串 ) )

示例：

示例 
timestamp = 1712800000000
method    = GET
uri       = /ncc/campaigns

签名目标 = "1712800000000.GET./ncc/campaigns"
X-Signature = Base64(HMAC-SHA256(SECRET_KEY, "1712800000000.GET./ncc/campaigns"))

curl实际请求示例

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: 1712800000000" \
  -H "X-API-KEY: {已申请的API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {生成的签名值}" \
  -H "Content-Type: application/json"

响应示例（200 OK）：

json 
[
  {
    "nccCampaignId": "cmp-001",
    "campaignTp": "WEB_SITE",
    "name": "品牌广告系列",
    "status": "ELIGIBLE",
    "dailyBudget": 100000
  }
]
💡 初学者提示：curl无需安装额外程序，可直接在Mac/Linux终端、Windows PowerShell中使用。
Java / PHP / C# 示例代码
其他语言的示例请参考GitBook官方文档（naver.github.io/searchad-apidoc）。

4 核心API工作流程 — 分步设置

推荐顺序：

前期准备 确认Business Channel ID
→
STEP 1 创建广告系列
→
STEP 2 创建广告组
→
STEP 3 注册广告素材
→
STEP 4 注册关键词

前期准备. 查询Business Channel — GET /ncc/channels

首先确认创建广告组时必需的pcChannelId / mobileChannelId值。

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/channels" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"
💡 从响应中记录channelTp为SITE的渠道的nccBusinessChannelId。此值将用于STEP 2的pcChannelId / mobileChannelId。

STEP 1. 创建广告系列 — POST /ncc/campaigns

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "campaignTp": "WEB_SITE",
    "name": "品牌广告系列",
    "dailyBudget": 100000,
    "useDailyBudget": true
  }'
参数说明示例值
campaignTp广告系列类型（Powerlink = WEB_SITE固定）"WEB_SITE"
name广告系列名称"品牌广告系列"
dailyBudget每日预算（韩元单位）100000
useDailyBudget是否使用每日预算true

响应中需要确认的内容：

json 
{
  "nccCampaignId": "cmp-abc123"
}

STEP 2. 创建广告组 — POST /ncc/adgroups

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-abc123",
    "name": "品牌关键词_组",
    "bidAmt": 1000,
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}"
  }'
💡 pcChannelId与mobileChannelId使用Business Channel API（GET /ncc/channels）查询获取的nccBusinessChannelId值。WEB_SITE广告系列中，请输入SITE类型的渠道ID。
参数说明备注
nccCampaignIdSTEP 1获取的广告系列ID必填
name广告组名称必填
pcChannelIdPC Business Channel ID必填 — GET /ncc/channels响应中的nccBusinessChannelId
mobileChannelId移动端Business Channel ID必填 — 可使用相同值
bidAmt基础出价（韩元，70~100,000）未填写时默认值为70

STEP 3. 注册广告素材 — POST /ncc/ads

素材内容需嵌套在"ad"对象内部传递，而非放在最顶层。

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/ads" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccAdgroupId": "grp-xyz456",
    "type": "TEXT_45",
    "ad": {
      "headline": "官方商城最高7折优惠",
      "description": "立即购买免运费。限今日特惠！",
      "pc": {
        "final": "https://www.brandstore.com/sale"
      },
      "mobile": {
        "final": "https://m.brandstore.com/sale"
      }
    }
  }'
参数说明限制
nccAdgroupId广告组ID（必填）从STEP 2获取
type素材类型（必填）"TEXT_45"固定
ad.headline广告标题1~15字
ad.description广告描述20~45字
ad.pc.finalPC目标URLhttps://或http://
ad.mobile.final移动端目标URLhttps://或http://

STEP 4. 注册关键词 — POST /ncc/keywords

nccAdgroupId通过URL查询参数传递，而非放在请求体中。请求体为关键词对象的数组。

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/keywords?nccAdgroupId=grp-xyz456" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "keyword": "耐克运动鞋",
      "bidAmt": 500,
      "useGroupBidAmt": false
    },
    {
      "keyword": "耐克Air Force折扣",
      "bidAmt": 300,
      "useGroupBidAmt": false
    }
  ]'
参数位置说明备注
nccAdgroupId查询参数要注册关键词的广告组ID从STEP 2获取
keywordBody要注册的关键词必填，可包含空格
bidAmtBody各关键词独立出价（70~100,000）useGroupBidAmt: false时生效
useGroupBidAmtBody是否使用广告组基础出价true时沿用广告组出价
linksBody各关键词独立着陆URL可选

💡 各关键词独立着陆URL设置

Powerlink可为每个关键词指定不同的着陆页。注册关键词时，在可选字段links中分别设置PC/移动端URL即可。未设置时，默认使用广告素材（Ad）中设置的URL。

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/keywords?nccAdgroupId=grp-xyz456" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "keyword": "耐克运动鞋特卖",
      "bidAmt": 500,
      "useGroupBidAmt": false,
      "links": {
        "pc": { "final": "https://www.brandstore.com/shoes/sale" },
        "mobile": { "final": "https://m.brandstore.com/shoes/sale" }
      }
    }
  ]'
💡 links的结构与STEP 3广告素材注册中ad.pc.final / ad.mobile.final的格式相同。

5 运营API — 查询·修改·统计

状态查询与管理

操作方法端点
查询所有广告系列GET/ncc/campaigns
查询特定广告系列GET/ncc/campaigns/{id}
修改广告系列PUT/ncc/campaigns/{id}
查询广告组GET/ncc/adgroups
查询关键词GET/ncc/keywords

广告系列暂停示例：

bash / curl 
curl -X PUT "https://api.searchad.naver.com/ncc/campaigns/{campaignId}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{"userLock": true}'

统计查询 — GET /stats

通过API获取广告效果数据的核心功能。

bash / curl 
curl -X GET "https://api.searchad.naver.com/stats?id={资源ID}&fields=[\"clkCnt\",\"impCnt\",\"salesAmt\"]&timeRange={\"since\":\"2026-04-01\",\"until\":\"2026-04-10\"}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"
指标字段名含义
impCnt展示次数
clkCnt点击次数
salesAmt广告费用
ctr点击率（CTR）
cpc单次点击费用（CPC）
ccnt转化次数
convAmt转化销售额
ror广告回报率（ROAS）

大量统计 — /stat-reports（报告方式）

查询周期较长或数据量较大时，使用异步报告方式：

步骤1 POST /stat-reports
创建报告请求
→ 获取reportId
→
步骤2 GET /stat-reports/{id}
检查状态
WAITING→RUNNING→DONE
→
步骤3 DONE状态
下载CSV文件

关键词搜索量确认 — GET /keywordstool

广告设置前通过API查询关键词搜索量的功能：

bash / curl 
curl -X GET "https://api.searchad.naver.com/keywordstool?hintKeywords=nikeShoes&showDetail=1" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

响应数据：

json 
{
  "keywordList": [
    {
      "relKeyword": "耐克运动鞋",
      "monthlyPcQcCnt": 12000,
      "monthlyMobileQcCnt": 45000,
      "compIdx": "high"
    }
  ]
}
字段含义
monthlyPcQcCnt每月PC搜索量
monthlyMobileQcCnt每月移动端搜索量
compIdx竞争程度（LOW / MEDIUM / HIGH）

出价估算 — GET /estimate/bid/keyword

关键词出价设置前，预先确认预估排名和费用：

bash / curl 
curl -X GET "https://api.searchad.naver.com/estimate/bid/keyword?keywords=nikeShoes&bids=500,1000,2000" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

6 广告附加信息API — POST /ncc/ad-extensions

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/ad-extensions" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "ownerId": "{nccAdId 或 nccAdgroupId}",
    "type": "SUB_LINKS",
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}",
    "userLock": false,
    "schedule": {}
  }'
参数说明备注
ownerId要关联广告附加信息的广告实体ID广告系列、广告组、素材ID均可（必填）
type广告附加信息类型代码必填
pcChannelIdPC Business Channel ID必填
mobileChannelId移动端Business Channel ID必填
userLock展示开/关（true=暂停）必填
schedule按星期和时间段的展示计划必填
⚠️ 广告附加信息内容（子链接URL等）的详细格式，请参考GitBook FAQ：github.com/naver/searchad-apidoc/wiki/FAQ
类型API type值
子链接SUB_LINKS
宣传文案DESCRIPTION
附加标题HEADLINE
电话号码PHONE
位置信息LOCATION

7 API运营技巧

共享预算运用 — /ncc/shared-budgets

将多个广告系列的预算集中管理为一个共享预算：

结构图 
POST /ncc/shared-budgets  → 创建共享预算
PUT  /ncc/campaigns/{id}  → 关联sharedBudgetId

多个广告系列共享一个预算池 → 高效消耗预算，集中管理

API调用注意事项

项目内容
Timestamp有效时间以服务器时间为准±300秒（5分钟）以内
请求限制（Rate Limit）每秒请求数量有限制——过多调用返回429错误
数组注册限制关键词等数组型POST时，每次最多建议100个
Customer IDX-Customer请求头为数字型广告账户ID

主要错误代码

HTTP代码含义解决方法
400错误的请求（参数错误）检查请求Body/参数
401认证失败重新确认Signature生成逻辑
403无权限确认Customer ID
404资源不存在确认ID值
429超出调用限制降低请求频率
500服务器错误稍后重试
