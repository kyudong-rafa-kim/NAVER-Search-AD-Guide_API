1 Differences from Powerlink API

💡 Authentication method, Base URL, and header structure are identical to Powerlink API. Shopping-specific settings begin at the campaign setup stage.
ItemPowerlinkShopping Search Ads
Base URLhttps://api.searchad.naver.comSame
AuthenticationHMAC-SHA256Same
campaignTpWEB_SITESHOPPING
Ad typeTEXT_45SHOPPING_PRODUCT_AD
Keyword registrationManual (POST /ncc/keywords)System auto-matching — no separate registration needed
Creative basisHeadline + description text entered directlyEP product data linked through CPS enrollment
💡 What is EP? EP (product feed) is linked during Naver Shopping CPS enrollment and contains product name, price, category, attributes, images, etc. Shopping Search Ads use the EP product data already linked during the enrollment process as ad creative. You do not need to connect a separate EP.
⚠️ Want to run a dedicated EP for Shopping Search Ads?
If you want to run Shopping Search Ads independently from the price comparison service, you can create a dedicated EP for Shopping Search Ads and use it separately.

2 Prerequisites — Business Channel & Product Group Query

Before setting up campaigns, you must first query the shopping channels and product groups linked to your ad account via API.

2-1. Business Channel Query — GET /ncc/channels

Query the shopping channels linked to your ad account (CPS enrollment completed).

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/channels" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

Response example:

json 
[
  {
    "nccBusinessChannelId": "ch-001",
    "channelTp": "SITE",
    "name": "My Shopping Mall",
    "pcInspectStatus": "APPROVED"
  }
]
⚠️ Only channels where pcInspectStatus is APPROVED can be used for ads.
Note the nccBusinessChannelId from the response — it is used as the pcChannelId / mobileChannelId value when creating an ad group.

2-2. Product Group Query — GET /ncc/product-groups

EP products linked during CPS enrollment are managed in Product Group units. Before creating ad groups, query the product group list to find the nccProductGroupId.

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/product-groups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

Response example:

json 
[
  {
    "nccProductGroupId": "pg-001",
    "nccBusinessChannelId": "ch-001",
    "mallId": "my-mall-001",
    "mallName": "My Shopping Mall",
    "name": "All Products Group",
    "brandName": "My Brand",
    "registeredProductType": "GENERAL",
    "registrationMethod": "WHOLE",
    "numberOfAdgroups": 0
  }
]
FieldDescription
nccProductGroupIdProduct group unique ID — used when creating ad groups
nccBusinessChannelIdLinked Business Channel ID
mallId / mallNameShopping mall identification
nameProduct group name
registeredProductTypeGENERAL (standard) / CATALOG / BLOG
registrationMethodWHOLE (all products) / EACH (selected products)
numberOfAdgroupsNumber of ad groups currently using this product group
💡 nccProductGroupId is used to specify the product group when creating an ad group.

3 Core API Workflow — Step-by-Step Setup

Recommended order for Shopping Search Ads API setup:

Pre-check Channel ID + Product Group ID
→
STEP 1 Create Campaign
→
STEP 2 Create Ad Group
→
STEP 3 Register Creative (Product)
💡 No keyword registration step — EP-based system handles automatic matching

STEP 1. Create Campaign — POST /ncc/campaigns

Uses the same endpoint as Powerlink. Only the campaignTp value differs — set to SHOPPING.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "campaignTp": "SHOPPING",
    "name": "ShoppingAds_Campaign",
    "dailyBudget": 100000,
    "useDailyBudget": true
  }'
ParameterDescriptionShopping value
campaignTpCampaign type"SHOPPING" (Powerlink uses "WEB_SITE")
nameCampaign nameFree input
dailyBudgetDaily budget (KRW)e.g., 100000
useDailyBudgetWhether to apply daily budgettrue

What to check in the response:

json 
{
  "nccCampaignId": "cmp-shop-001"
}

STEP 2. Create Ad Group — POST /ncc/adgroups

The grouping strategy varies based on SKU volume.

Case 1 — Category-based grouping (large SKU)

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-shop-001",
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}",
    "nccProductGroupId": "pg-001",
    "name": "Category1_Group",
    "bidAmt": 200
  }'

Case 2 — Core keyword-based grouping (small SKU)

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-shop-001",
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}",
    "nccProductGroupId": "pg-001",
    "name": "CoreKeyword1_Group",
    "bidAmt": 200
  }'
ParameterDescriptionNotes
nccCampaignIdCampaign ID from STEP 1Required
pcChannelIdPC Business Channel IDRequired — nccBusinessChannelId from CH 2-1 response
mobileChannelIdMobile Business Channel IDRequired — same value can be used
nccProductGroupIdProduct group IDnccProductGroupId from CH 2-2 response
nameAd group nameName according to your grouping strategy
bidAmtBase bid (KRW, 70–100,000)Individual creative bids can also be set

What to check in the response:

json 
{
  "nccAdgroupId": "grp-shop-001"
}

STEP 3. Register Creative (Product) — POST /ncc/ads

Shopping Search Ad creatives are based on EP products linked during CPS enrollment. Specify type as SHOPPING_PRODUCT_AD. Set a per-creative bid via adAttr.bidAmt; to use the group's base bid, set useGroupBidAmt: true.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/ads" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccAdgroupId": "grp-shop-001",
    "type": "SHOPPING_PRODUCT_AD",
    "adAttr": {
      "bidAmt": 200,
      "useGroupBidAmt": false
    }
  }'
ParameterDescriptionNotes
nccAdgroupIdAd group IDObtained from STEP 2
typeCreative type"SHOPPING_PRODUCT_AD" fixed (see GitBook, GET /ncc/product-groups)
adAttr.bidAmtPer-creative individual bidUsed when setting an individual creative bid
adAttr.useGroupBidAmtWhether to use the group's base bidtrue follows the group bid (see GitBook)
💡 Keywords are not registered separately. The system automatically matches relevant keywords based on the EP product's name, attributes, and category information.
⚠️ Detailed custom fields for creative details (such as additional promotional copy) are not separately documented for the SHOPPING_PRODUCT_AD type ad object schema in the GitBook Reference. For detailed implementation, refer to the official Naver Search Ads API guide.

Statistics Query — GET /stats

Uses the same endpoint as Powerlink.

bash / curl 
curl -X GET "https://api.searchad.naver.com/stats?id={campaignId or groupId or adId}&fields=[\"clkCnt\",\"impCnt\",\"salesAmt\",\"convCnt\"]&timeRange={\"since\":\"2026-04-01\",\"until\":\"2026-04-10\"}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

Key metrics for Shopping Search Ads:

Metric fieldMeaningWhy it matters for Shopping
impCntImpressionsCheck auto-matching coverage
clkCntClicksAssess creative CTR
salesAmtAd spendConfirm budget consumption
ctrClick-through rateGauge creative attractiveness
convCntConversionsCheck connection to actual purchases
convAmtConversion revenueBasis for ROAS calculation

Bulk Statistics — /stat-reports

Use the asynchronous report method for long date ranges or large data volumes:

Flow 
Step 1. POST /stat-reports  → Create report request → Obtain reportId
Step 2. GET  /stat-reports/{reportId}  → Check status (WAITING → RUNNING → DONE)
Step 3. DONE status → Download CSV file from the report download URL

6 Operations Tips — Shopping Search Ads Specific

Creative Quality Determines Exposure Range

Since Shopping Search Ads don't require manual keyword registration, the quality of product information determines the exposure range:

  • Include rich core keywords in the product name → automatically matched to more keywords
  • Set the category accurately → improves accuracy of relevant query matching
  • Always fill in the additional promotional text → fastest way to improve CTR

Manage Bids Separately by Group

For large SKU catalogs, set different bids per group to run budgets efficiently:

bash / curl 
# High-margin category group → higher bid
curl -X PUT "https://api.searchad.naver.com/ncc/adgroups/{highMarginGroupId}" \
  -d '{"bidAmt": 500}'

# Regular category group → lower bid
curl -X PUT "https://api.searchad.naver.com/ncc/adgroups/{regularGroupId}" \
  -d '{"bidAmt": 150}'
