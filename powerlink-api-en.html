1 Prerequisites — API License Issuance

Required Conditions Checklist

ItemDetails
Naver ad accountAt least one Site Search Ads (Powerlink) campaign must exist
API licenseMust be separately applied for within the ad system
Issued credentialsAPI KEY (X-API-KEY), SECRET KEY (for signature generation), Customer ID (X-Customer)

API License Application Steps

  1. Log in to Naver Ad System (searchad.naver.com)
  2. Click [My Info] → [API Management] in the top right
  3. Click the [Apply for API License] button
  4. Agree to the Terms of Service and complete the application
  5. Check and save your API KEY / SECRET KEY on the screen
⚠️ The SECRET KEY is shown only once at issuance. Save it in a secure location immediately.

ItemValue

Overall Request Structure

Structure 
https://api.searchad.naver.com/{resource path}
  └── HTTP Method: GET / POST / PUT / DELETE
  └── Headers: 4 authentication headers (required)
  └── Body: JSON (for POST/PUT requests)

3 Authentication

4 Required Headers

All API requests must include the following 4 headers:

HeaderDescription
X-TimestampCurrent time (Unix timestamp, milliseconds)
X-API-KEYYour issued API KEY
X-CustomerAdvertiser Customer ID
X-SignatureHMAC-SHA256 signature value

Signature Generation Formula

Signature Formula 
Signing string = {timestamp} + "." + {HTTP_METHOD} + "." + {request_uri}

X-Signature = Base64( HMAC-SHA256( SECRET_KEY, signing string ) )

Example:

Example 
timestamp = 1712800000000
method    = GET
uri       = /ncc/campaigns

Signing string = "1712800000000.GET./ncc/campaigns"
X-Signature = Base64(HMAC-SHA256(SECRET_KEY, "1712800000000.GET./ncc/campaigns"))

curl Request Example

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: 1712800000000" \
  -H "X-API-KEY: {your API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {generated signature}" \
  -H "Content-Type: application/json"

Response example (200 OK):

json 
[
  {
    "nccCampaignId": "cmp-001",
    "campaignTp": "WEB_SITE",
    "name": "Brand Campaign",
    "status": "ELIGIBLE",
    "dailyBudget": 100000
  }
]
💡 Beginner tip: curl can be used directly in Mac/Linux Terminal and Windows PowerShell without installing any additional programs.

Pre-check. Business Channel Query — GET /ncc/channels

First confirm the pcChannelId / mobileChannelId values required for creating an ad group.

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/channels" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"
💡 From the response, note the nccBusinessChannelId of the channel where channelTp is SITE. This value is used for pcChannelId / mobileChannelId in STEP 2.

STEP 1. Create Campaign — POST /ncc/campaigns

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "campaignTp": "WEB_SITE",
    "name": "My Brand Campaign",
    "dailyBudget": 100000,
    "useDailyBudget": true
  }'
ParameterDescriptionExample
campaignTpCampaign type (Powerlink = WEB_SITE fixed)"WEB_SITE"
nameCampaign name"Brand Campaign"
dailyBudgetDaily budget (in KRW)100000
useDailyBudgetWhether to apply daily budgettrue

What to check in the response:

json 
{
  "nccCampaignId": "cmp-abc123"
}

STEP 2. Create Ad Group — POST /ncc/adgroups

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-abc123",
    "name": "BrandKeyword_Group",
    "bidAmt": 1000,
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}"
  }'
💡 pcChannelId and mobileChannelId use the nccBusinessChannelId value obtained from the Business Channel API (GET /ncc/channels). For WEB_SITE campaigns, enter the SITE type channel ID.
ParameterDescriptionNotes
nccCampaignIdCampaign ID from STEP 1Required
nameAd group nameRequired
pcChannelIdPC Business Channel IDRequired — nccBusinessChannelId from GET /ncc/channels
mobileChannelIdMobile Business Channel IDRequired — same value as PC can be used
bidAmtBase bid (KRW, 70–100,000)Default 70 if omitted

STEP 3. Register Ad Creative — POST /ncc/ads

Creative content is nested inside the "ad" object, not at the top level.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/ads" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccAdgroupId": "grp-xyz456",
    "type": "TEXT_45",
    "ad": {
      "headline": "Official Store — Up to 30% Off",
      "description": "Free shipping on purchase now. Today\'s special price!",
      "pc": {
        "final": "https://www.brandstore.com/sale"
      },
      "mobile": {
        "final": "https://m.brandstore.com/sale"
      }
    }
  }'
ParameterDescriptionConstraint
nccAdgroupIdAd group ID (Required)Obtained from STEP 2
typeCreative type (Required)"TEXT_45" fixed
ad.headlineAd headline1–15 characters
ad.descriptionAd description20–45 characters
ad.pc.finalPC landing URLhttps:// or http://
ad.mobile.finalMobile landing URLhttps:// or http://

STEP 4. Register Keywords — POST /ncc/keywords

nccAdgroupId is passed as a URL query parameter, not in the request body. The body is an array of keyword objects.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/keywords?nccAdgroupId=grp-xyz456" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "keyword": "Nike sneakers",
      "bidAmt": 500,
      "useGroupBidAmt": false
    },
    {
      "keyword": "Nike Air Force discount",
      "bidAmt": 300,
      "useGroupBidAmt": false
    }
  ]'
ParameterLocationDescriptionNotes
nccAdgroupIdQuery paramAd group ID to register keywords intoObtained from STEP 2
keywordBodyKeyword to registerRequired; spaces allowed
bidAmtBodyPer-keyword bid (70–100,000 KRW)Applied when useGroupBidAmt: false
useGroupBidAmtBodyWhether to use the group's base bidtrue follows the group bid
linksBodyPer-keyword individual landing URLOptional

💡 Per-Keyword Individual Landing URL

Powerlink allows you to specify a different landing page for each keyword. When registering keywords, set the optional links field with separate PC/mobile URLs. If not set, the URL from the ad creative is used by default.

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/keywords?nccAdgroupId=grp-xyz456" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "keyword": "Nike sneakers sale",
      "bidAmt": 500,
      "useGroupBidAmt": false,
      "links": {
        "pc": { "final": "https://www.brandstore.com/shoes/sale" },
        "mobile": { "final": "https://m.brandstore.com/shoes/sale" }
      }
    }
  ]'
💡 The structure of links follows the same pattern as ad.pc.final / ad.mobile.final in STEP 3 Ad Creative registration.

Statistics Query — GET /stats

Retrieve ad performance data via API.

bash / curl 
curl -X GET "https://api.searchad.naver.com/stats?id={resourceId}&fields=[\"clkCnt\",\"impCnt\",\"salesAmt\"]&timeRange={\"since\":\"2026-04-01\",\"until\":\"2026-04-10\"}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

7 API Operations Tips

Shared Budget — /ncc/shared-budgets

Centrally manage multiple campaign budgets as a single shared budget pool:

Structure 
POST /ncc/shared-budgets  → Create shared budget
PUT  /ncc/campaigns/{id}  → Link sharedBudgetId

Multiple campaigns share one budget pool → efficient budget spending, managed from one place

API Call Precautions

ItemDetails
Timestamp validityWithin ±300 seconds (5 minutes) of server time
Rate limitPer-second request limit applies — excessive calls return 429 error
Array batch limitUp to 100 items per request recommended for array POST (e.g., keywords)
Customer IDX-Customer header is the numeric ad account ID

Common Error Codes
