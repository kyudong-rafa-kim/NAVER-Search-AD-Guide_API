1 与Powerlink API的区别

💡 认证方式、Base URL、请求头结构与Powerlink API完全相同。从广告系列设置阶段开始，需要使用购物搜索广告专属配置。
项目Powerlink购物搜索广告
Base URLhttps://api.searchad.naver.com相同
认证方式HMAC-SHA256相同
campaignTpWEB_SITESHOPPING
Ad typeTEXT_45SHOPPING_PRODUCT_AD
关键词注册手动注册 (POST /ncc/keywords)系统自动匹配 — 无需单独注册
素材来源直接输入标题+描述文本利用CPS入驻时关联的EP商品
💡 什么是EP？ EP是Naver购物CPS入驻时关联的商品数据源（商品名·价格·类目·属性·图片等）。购物搜索广告利用入驻过程中已关联的EP商品数据作为广告素材，无需单独重新关联EP。
⚠️ 如需单独运营购物搜索广告专用EP？
若希望将购物搜索广告与价格比较独立运营， 可选择单独创建购物搜索广告专用EP来使用。

2 前期准备 — Business Channel & Product Group查询

在设置广告系列之前，需先通过API查询与广告账户关联的购物频道和商品组。

2-1. Business Channel查询 — GET /ncc/channels

查询与广告账户关联的购物频道（已完成CPS入驻状态）。

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/channels" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

响应示例：

json 
[
  {
    "nccBusinessChannelId": "ch-001",
    "channelTp": "SITE",
    "name": "我的购物商城",
    "pcInspectStatus": "APPROVED"
  }
]
⚠️ 只有pcInspectStatus为APPROVED的频道才能用于投放广告。
请记录响应中的nccBusinessChannelId——创建广告组时将用作pcChannelId / mobileChannelId的值。

2-2. Product Group查询 — GET /ncc/product-groups

CPS入驻时关联的EP商品以商品组（Product Group）为单位进行管理。创建广告组前，需查询账户中已注册的商品组列表，确认nccProductGroupId。

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/product-groups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

响应示例：

json 
[
  {
    "nccProductGroupId": "pg-001",
    "nccBusinessChannelId": "ch-001",
    "mallId": "my-mall-001",
    "mallName": "我的购物商城",
    "name": "全部商品组",
    "brandName": "My Brand",
    "registeredProductType": "GENERAL",
    "registrationMethod": "WHOLE",
    "numberOfAdgroups": 0
  }
]
字段说明
nccProductGroupId商品组唯一ID — 创建广告组时使用
nccBusinessChannelId关联的Business Channel ID
mallId / mallName购物商城标识信息
name商品组名称
registeredProductTypeGENERAL（普通商品）/ CATALOG（目录）/ BLOG
registrationMethodWHOLE（全部注册）/ EACH（选择注册）
numberOfAdgroups正在使用该商品组的广告组数量
💡 nccProductGroupId用于创建广告组时指定关联的商品组。

3 核心API工作流程 — 分步设置

购物搜索广告API设置推荐顺序：

前期查询 确认Channel ID + Product Group ID
→
STEP 1 创建广告系列
→
STEP 2 创建广告组
→
STEP 3 注册素材（商品）
💡 无关键词注册步骤 — 基于EP的系统自动匹配

STEP 1. 创建广告系列 — POST /ncc/campaigns

与Powerlink使用相同的接口端点。仅campaignTp值不同，需设为SHOPPING。

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/campaigns" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "campaignTp": "SHOPPING",
    "name": "购物搜索广告_系列",
    "dailyBudget": 100000,
    "useDailyBudget": true
  }'
参数说明购物设置值
campaignTp广告系列类型"SHOPPING"（Powerlink为"WEB_SITE"）
name广告系列名称自由填写
dailyBudget日预算（韩元）例：100000
useDailyBudget是否启用日预算true

响应中需确认的内容：

json 
{
  "nccCampaignId": "cmp-shop-001"
}

STEP 2. 创建广告组 — POST /ncc/adgroups

根据SKU规模，分组策略有所不同。

Case 1 — 按类目分组（SKU较多）

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-shop-001",
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}",
    "nccProductGroupId": "pg-001",
    "name": "类目1_广告组",
    "bidAmt": 200
  }'

Case 2 — 按核心关键词分组（SKU较少）

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/adgroups" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccCampaignId": "cmp-shop-001",
    "pcChannelId": "{nccBusinessChannelId}",
    "mobileChannelId": "{nccBusinessChannelId}",
    "nccProductGroupId": "pg-001",
    "name": "核心关键词1_广告组",
    "bidAmt": 200
  }'
参数说明备注
nccCampaignIdSTEP 1获取的广告系列ID必填
pcChannelIdPC业务频道ID必填 — 第2-1章响应中的nccBusinessChannelId
mobileChannelId移动业务频道ID必填 — 可使用相同值
nccProductGroupId商品组ID第2-2章响应中的nccProductGroupId
name广告组名称根据分组策略命名
bidAmt默认出价（韩元，70~100,000）也可按素材单独设置

响应中需确认的内容：

json 
{
  "nccAdgroupId": "grp-shop-001"
}

STEP 3. 注册素材（商品）— POST /ncc/ads

购物搜索广告的素材基于CPS入驻时关联的EP商品创建。将type指定为SHOPPING_PRODUCT_AD进行注册。素材级别的出价通过adAttr中的bidAmt设置；如需沿用广告组默认出价，请将useGroupBidAmt设为true。

bash / curl 
curl -X POST "https://api.searchad.naver.com/ncc/ads" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "nccAdgroupId": "grp-shop-001",
    "type": "SHOPPING_PRODUCT_AD",
    "adAttr": {
      "bidAmt": 200,
      "useGroupBidAmt": false
    }
  }'
参数说明备注
nccAdgroupId广告组ID在STEP 2中获取
type素材类型固定为"SHOPPING_PRODUCT_AD"（请查阅GitBook，GET /ncc/product-groups）
adAttr.bidAmt素材级别个别出价设置个别出价时使用
adAttr.useGroupBidAmt是否使用广告组默认出价设为true时沿用广告组出价（请查阅GitBook）
💡 无需单独注册关键词。系统将根据EP商品的素材名称、属性、类目信息自动匹配相关关键词。
⚠️ 附加推广文案等素材详细自定义字段，在GitBook Reference中未单独列出SHOPPING_PRODUCT_AD类型的ad对象schema文档。详细实现时请参考Naver搜索广告API官方指南。

4 素材查询与修改

素材全量查询 — GET /ncc/ads

bash / curl 
curl -X GET "https://api.searchad.naver.com/ncc/ads?nccAdgroupId=grp-shop-001" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

素材出价修改 — PUT /ncc/ads/{nccAdId}

bash / curl 
curl -X PUT "https://api.searchad.naver.com/ncc/ads/{nccAdId}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{
    "adAttr": {
      "bidAmt": 300,
      "useGroupBidAmt": false
    }
  }'

5 运营API — 查询·修改·统计

广告系列/广告组状态管理

操作Method接口端点
广告系列全量查询GET/ncc/campaigns
暂停广告系列PUT/ncc/campaigns/{id}
广告组查询GET/ncc/adgroups
广告组出价修改PUT/ncc/adgroups/{id}
素材查询GET/ncc/ads
素材修改PUT/ncc/ads/{id}

暂停广告系列示例：

bash / curl 
curl -X PUT "https://api.searchad.naver.com/ncc/campaigns/{campaignId}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{"userLock": true}'

广告组出价修改示例：

bash / curl 
curl -X PUT "https://api.searchad.naver.com/ncc/adgroups/{adgroupId}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}" \
  -H "Content-Type: application/json" \
  -d '{"bidAmt": 300}'

数据统计查询 — GET /stats

与Powerlink使用相同的接口端点。

bash / curl 
curl -X GET "https://api.searchad.naver.com/stats?id={广告系列ID或广告组ID或素材ID}&fields=[\"clkCnt\",\"impCnt\",\"salesAmt\",\"convCnt\"]&timeRange={\"since\":\"2026-04-01\",\"until\":\"2026-04-10\"}" \
  -H "X-Timestamp: {timestamp}" \
  -H "X-API-KEY: {API KEY}" \
  -H "X-Customer: {Customer ID}" \
  -H "X-Signature: {signature}"

购物搜索广告中需重点关注的指标：

指标字段名含义在购物中的重要原因
impCnt展示次数确认自动匹配范围
clkCnt点击次数判断素材CTR
salesAmt广告费用确认预算消耗
ctr点击率判断素材吸引力
convCnt转化次数确认是否带来实际购买
convAmt转化销售额ROAS计算基础

大量数据统计 — /stat-reports

当查询时间跨度较长或数据量较大时，使用异步报告方式：

流程 
Step 1. POST /stat-reports  → 请求生成报告 → 获取reportId
Step 2. GET  /stat-reports/{reportId}  → 确认状态 (WAITING → RUNNING → DONE)
Step 3. DONE状态 → 从报告下载URL下载CSV文件

6 API运营技巧 — 购物搜索广告专属

素材质量决定展示范围

购物搜索广告无需手动注册关键词，因此商品信息的质量决定了展示范围：

  • 商品名称中包含丰富的核心关键词 → 自动匹配到更多关键词
  • 精准设置类目 → 提升相关搜索词的匹配精度
  • 始终填写附加推广文案 → 提升CTR最快速的方法

分组出价差异化管理

SKU较多时，对不同广告组设置差异化出价，实现预算的高效运营：

bash / curl 
# 高利润类目广告组 → 较高出价
curl -X PUT "https://api.searchad.naver.com/ncc/adgroups/{高利润组ID}" \
  -d '{"bidAmt": 500}'

# 普通类目广告组 → 较低出价
curl -X PUT "https://api.searchad.naver.com/ncc/adgroups/{普通组ID}" \
  -d '{"bidAmt": 150}'

API调用注意事项

项目内容
Timestamp有效时间以服务器时间为准，±300秒（5分钟）以内
请求限制（Rate Limit）有每秒请求数限制 — 调用过于频繁时返回429错误
数组注册上限数组形式POST素材时，单次最多推荐100个
productId确认在购物合作中心确认准确的商品ID后再使用

主要错误代码

HTTP代码含义解决方法
400错误请求（参数错误）检查campaignTp、productId等参数
401认证失败重新确认Signature生成逻辑
403权限不足确认Channel关联状态（APPROVED）
404资源不存在确认ID值
429超出调用限制降低请求频率
500服务器错误稍后重试
